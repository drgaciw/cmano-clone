Project Aegis — Catalog Logical Data Model

UML-style class diagram of the SQLite game catalog (Req 06 Database Intelligence). Entities map 1:1 to physical tables under assets/data/catalog/. Logical associations are shown even where SQLite foreign keys are sparse (app/write-gate enforced).

Source DB: baltic_patrol.db + migrations 001–011 Schema version constant: 010 (latest migration: 011) Generated for data-architecture review

4. Key relationships (kill chain)

Detect Platform 1—* Sensor
Approved sensors with base_pd form the detect leg. Sensor-as-pseudo-platform imports break this logical FK.
Host Platform 1—* PlatformMount
Mount types: VLS, rail, gun, tube, pylon. Review gate filters dependency edges.
Configure Platform 1—* PlatformLoadout
Role-tagged packages (ASUW / AAW / ASW / strike / recon).
Engage edge Magazine *(Platform, Loadout, Mount, Weapon)*
Quantity + reload. Builds PlatformToMountToWeapon dependency edges.
Weapon envelope WeaponCatalog 1—* Magazine
min/max range meters consumed by catalog intelligence & engage DLZ path.
C2 Platform *—* LinkCatalog via PlatformComms
Latency and role (tx/rx/relay) for data-link doctrine.
Release train Snapshot 1—* DbRelease
Content hash + TL branch metadata for export/diff.
Write gate StagingBatch 1—* staging_* tables
Only SQL FKs in the model; live integrity is app-validated.

5. Entity catalog — labels & descriptions

Entity / table Package Label Description Primary key
platform Core Platform Unit or facility identity within a snapshot: placement, domain, class, combat radius, nationality, TL. (platform_id, snapshot_id)
catalog_snapshot Governance Catalog Snapshot Named immutable catalog cut with content hash and technology-level branch tag. snapshot_id
db_release Governance DB Release Versioned release of a snapshot for train/diff/rollback (schema version + notes). release_version
sensor Engage Sensor Binding Platform-mounted sensor with detection scale (base_pd) and full P0 provenance (TRL, confidence, citation, review). (platform_id, sensor_id)
sensor_quarantine Quarantine Sensor Quarantine Sensors held out of live export pending evidence or reject reason. (platform_id, sensor_id)
weapon_catalog Engage Weapon Type Shared weapon definition: display name, range envelope, type, guidance. weapon_id
link_catalog Engage Link Medium Communications / data-link type with nominal latency. link_id
platform_mount Engage Mount Weapon host station on a platform (arc, capacity, mount type, review state). (platform_id, mount_id)
platform_loadout Engage Loadout Named package of magazines for a mission role; may be default loadout. (platform_id, loadout_id)
platform_magazine Engage Magazine / Fitting Quantity of a weapon on a mount within a loadout; core kill-chain edge. (platform_id, loadout_id, mount_id, weapon_id)
platform_comms Engage Comms Fitting Platform association to a link medium with TX/RX role and provenance. (platform_id, link_id)
platform_mobility Phase B Mobility Profile Speed, altitude/depth, fuel, range, endurance for kinematic planning. platform_id
platform_signature Phase B Signature Profile RCS, IR, acoustic, magnetic detectability parameters. platform_id
platform_emcon Phase B EMCON Posture Emission-control condition and emitter posture for a platform. (platform_id, condition, emitter_id)
platform_damage Phase B Damage Model HP budget, withdraw threshold, critical flags for survivability. platform_id
catalog_staging_batch Staging Staging Batch Propose unit for write-gate: actor, rationale, approval state, record count. batch_id
catalog_staging_* Staging Staging Rows Pending copies of sensor, platform, weapon, mount, loadout, magazine, comms, link, mobility, signature, EMCON, damage. (batch_id, …entity keys)
catalog_change_log Governance Change Log Append-only audit of approved field mutations (actor, before/after, release). change_id
