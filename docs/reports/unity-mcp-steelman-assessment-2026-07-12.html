3. Steelman FOR Unity-MCP

  1. Closes the agentic gap Unity actually has. Sim/delegation is headless-complete. Remaining work is Editor-native (UXML/USS, PanelHosts, scenes, assets, Game View smoke). Unity-MCP is the wired path for agents to drive that layer.
  2. Matches the integration review’s highest-leverage pain. unity-integration-review-2026-07-07.md §4.2–4.4 — MCP skills already cover scene/gameobject/assets/screenshot/profiler loops Cloud cannot run.
  3. Skill surface is already paid for. 77 skills + README safety rules — adoption is activate session, not invent tooling. No need for a separate unity-mcp-operator agent.
  4. Low risk to Release invariants if bounded. Editor/dev tooling only; headless gates stay authoritative; DelegationBridge zero-touch stays in skill preambles.
  5. Mil-sim C2 needs visual truth. Screenshots + console + hierarchy beats markdown-only Unity forever.
