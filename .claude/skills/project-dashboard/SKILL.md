# Project Dashboard

Producer agent workflow that aggregates production, design, architecture, asset,
GitNexus, and audit data into a living dashboard at `docs/reports/project-dashboard.md`.

**This skill may write report files only.** Do not modify source code.

---

## When to Run

- Manual: `/project-dashboard` or `/project-dashboard pm`
- Scheduled: Cursor Automation at 09:00 and 21:00 (see `.cursor/automations/README.md`)
- After major merges when leadership needs an updated snapshot

---

## 1. Determine Run Label

- Argument `$ARGUMENTS[0]`: `am` or `pm` if provided
- Otherwise: local hour `< 12` → `am`, else `pm`
- Timestamp: ISO 8601 in report header

---

## 2. GitNexus Polling

Run from repository root. Prefer CLI when MCP is unavailable.

```bash
npx gitnexus status
# If stale or missing index:
npx gitnexus analyze
npx gitnexus detect-changes
```

Optional watchlist impact (report risk levels only — do not edit symbols):

```bash
npx gitnexus impact IRoeFilter -d upstream
npx gitnexus impact DecisionLog -d upstream
npx gitnexus impact DelegationOrchestrator -d upstream
npx gitnexus impact SimTickPipeline -d upstream
```

When GitNexus MCP is available, prefer:

- `gitnexus://repo/cmano-clone/context` — counts, staleness, indexed commit
- `gitnexus_detect_changes()` — changed symbols and affected processes
- `gitnexus_query({query: "..."})` — cluster/process activity

Capture: indexed commit, node/symbol count, edge/relationship count, cluster count,
flow/process count, index status, changed symbols, affected processes, watchlist risks.

---

## 3. Read Previous State

Read `production/dashboard-state.yaml` if it exists. Use it for the **Since Last Update**
delta section. If missing, label this run as **baseline**.

---

## 4. Filesystem Aggregation

Scan and count (Glob + Read as needed):

| Area | Paths |
|------|-------|
| Sprints | `production/sprints/`, `production/sprint-status.yaml` |
| Milestones | `production/milestones/` |
| Epics/stories | `production/epics/` |
| GDDs | `design/gdd/` — cross-check `design/gdd/systems-index.md` |
| Architecture | `docs/architecture/` ADRs + `architecture.md` status |
| Assets | `design/assets/asset-manifest.md` |
| Audits | `production/determinism/*.md` (latest) |
| Plans | `docs/superpowers/plans/*.md` — count `- [x]` vs `- [ ]` |
| Tests | `src/*Tests/**/*.cs` |
| Source | `src/**/*.cs` (exclude Tests) |

**Honesty rules:**

- Never invent sprint completion ratios when `production/sprints/` is empty
- Never assign milestone deadlines without a source document
- Label proxy metrics as "Proxy" or "Inferred"

---

## 5. Write Dashboard

Update `docs/reports/project-dashboard.md` using structure from
`.claude/docs/templates/project-stage-report.md` plus these sections:

1. Header (Generated, Last Updated, Run label, Stage)
2. Executive Summary
3. Since Last Update (GitNexus + Git) — skip deltas on baseline
4. GitNexus Code Intelligence
5. Sprint Status
6. Milestone Tracking
7. Completeness Overview (Design, Architecture, Production, Source/Tests)
8. Asset Manifest
9. Gaps Identified (Critical / Important / Nice-to-Have)
10. Recommended Next Steps
11. Follow-Up Skills to Run
12. Appendix: File Counts

Footer:

`*Generated by producer agent — aggregated from production, design, architecture, GitNexus, and audit artifacts*`

---

## 6. Archive Snapshot

Copy the dashboard to:

`docs/reports/dashboard-snapshots/YYYY-MM-DD-{am|pm}.md`

---

## 7. Update State File

Write `production/dashboard-state.yaml` with:

```yaml
last_run:
  timestamp: "<ISO8601>"
  label: am|pm
gitnexus:
  indexed_commit: "<sha>"
  nodes: <int>
  edges: <int>
  clusters: <int>
  flows: <int>
  index_status: fresh|stale
  changed_symbols_count: <int>
  affected_processes_count: <int>
filesystem:
  sprint_plans: <int>
  milestones: <int>
  gdd_systems_with_docs: <int>
  gdd_systems_total: 20
  adrs: <int>
  test_files: <int>
  source_files: <int>
```

## 8. Optional PR

If material changes since last run (new symbols, new sprint stories, new blockers,
new GDDs, or first baseline):

- Open PR: `chore(dashboard): daily update YYYY-MM-DD {am|pm}`
- Include only `docs/reports/**`, `production/dashboard-state.yaml`
